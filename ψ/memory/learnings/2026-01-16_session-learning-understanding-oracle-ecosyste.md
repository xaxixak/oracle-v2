# ## Session Learning: Understanding Oracle Ecosystem

## Session Learning: Understanding Oracle Ecosystem

### Context
คุยกับ user เรื่อง Issue #6 และพยายามเข้าใจว่า Oracle agents คุยกันยังไง

### Key Realizations

1. **ไม่มี magic** - Oracle v2 ไม่มีระบบ agent-to-agent อัตโนมัติ
2. **Social convention** - การ "คุยกัน" คือมนุษย์สั่ง Claude → post ไป GitHub
3. **Shared learning space** - Issue #6 เป็นที่แชร์ learnings ไม่ใช่ chat system
4. **Naming is optional** - ชื่อ Le, Arthur ตั้งโดยมนุษย์ ไม่มีใน system

### User's Insight
User ถูกต้องที่สังเกตว่า:
- "มันคือการ learning แล้วแบ่ง learning"
- ไม่ใช่ automated agent communication

### What Oracle v2 Actually Does
- `oracle_learn` - บันทึก patterns/learnings
- `oracle_search` - ค้นหา knowledge
- `oracle_thread` - internal discussion (ไม่ได้ไปถึง Oracle อื่น)
- `oracle_trace` - บันทึก discovery sessions

### What It Doesn't Do (Yet)
- ❌ Cross-instance messaging
- ❌ Auto-notify other Oracles
- ❌ Webhook/automation for GitHub comments
